## 批发站 P0（MVP）可执行需求说明

> 文档时间：2026-03-14  
> 依据文档：`docs/prd-wholesale-station-template.md`、`docs/wholesale-site-and-mobigo-seed.md`  
> 范围：仅覆盖 P0/MVP（零售商可登录、看价、下单，订单写入现有仓库系统）

---

## 一、P0 范围总览

### 1.1 页面与功能清单（仅列 P0）

- **首页（P0）**
  - 展示：导航入口、部分类目、部分商品推荐（无价格，无库存）。
  - 支持：点击类目进入「商品列表」、点击商品进入「商品详情」。
  - 登录后首页可继续使用，显示内容与未登录一致或略有差异，但**不强依赖**首页完成登录后才可下单（登录后可直接从类目/搜索进列表）。

- **类目导航（P0）**
  - 与仓库系统类目一致（含 Mobigo 类目）。
  - 左侧/顶部树形展示，支持展开/收起，点击类目进入该类目的「商品列表」。

- **商品列表页（P0）**
  - 按类目浏览商品。
  - 支持关键词搜索、简单排序（至少价格排序，若实现复杂可先只做「默认排序」）。
  - 未登录：只显示商品名称、主图、基础规格，**不显示价格、库存、起订量**。
  - 已登录：显示批发价、库存状态（有货/缺货/少量）、起订量入口、加入购物车按钮。

- **商品详情页（P0）**
  - 展示：多图、名称、中英文描述、关键规格。
  - 未登录：隐藏价格、真实库存、起订量；提供「登录/注册查看价格」入口。
  - 已登录：显示批发价、起订量、库存状态、可选数量、加入购物车按钮。

- **注册/登录页（P0）**
  - 注册：支持零售商在线注册（提交公司名、联系人、电话、邮箱、密码等基础信息）。注册后默认生成一个「客户」记录。
  - 登录：邮箱 + 密码（或你现有的登录字段）；登录成功后返回 JWT 等现有认证信息。
  - 忘记密码：本期 P0 可选做，若实现成本较高可标记为 P1。

- **购物车页（P0）**
  - 列出当前用户购物车中的所有 SKU：商品名、规格、单价、数量、小计。
  - 支持修改数量、删除行。
  - 在提交前校验：起订量规则（按 SKU 或整单，见 5.2）。
  - 展示订单金额汇总（不含运费规则，P0 可简单处理）。

- **下单与订单列表（P0）**
  - 提交订单：从购物车生成销售订单，写入现有仓库/销售系统。
  - 订单列表：展示当前登录客户自己的订单列表，字段含：订单号、下单时间、金额、订单状态（待确认/拣货中/已发货/已完成/已取消）。
  - 订单详情：展示订单行（SKU、数量、单价、小计）和基础物流信息（P0 可先预留字段，显示简单文本）。

- **基础后台支持（P0）**
  - 可选：不新建完整后台 UI，P0 阶段以**接口为主**，运营/销售通过现有后台或数据库查看订单。
  - 必要能力：保证新来源的订单可以在现有系统中区分「来源＝批发站」，方便后续统计。

> 非 P0：采购意图报表、销量报表、补货建议、清仓/促销后台配置、客服后台等全部归类为 P1/P2，仅在接口设计时适当预留字段，不在本说明展开。

## 二、页面与路由结构（P0）

> 具体技术栈（React/Next/Vue 等）可在实现阶段再细化，这里从「URL 与功能」角度定义。

- `/`  
  - 组件：`WholesaleHome`  
  - 功能：展示基础导航、推荐类目和商品；点击进入列表/详情。

- `/categories/:categoryId`  
  - 组件：`WholesaleProductList`  
  - 功能：指定类目的商品列表；支持 query 参数：`?q=keyword&sort=xxx&page=1&pageSize=20`。

- `/products/:productId`  
  - 组件：`WholesaleProductDetail`  
  - 功能：商品详情；根据登录态展示不同信息（有价/无价）。

- `/login`  
  - 组件：`WholesaleLogin`  
  - 功能：零售商登录。

- `/register`  
  - 组件：`WholesaleRegister`  
  - 功能：零售商注册。

- `/cart`  
  - 组件：`WholesaleCart`  
  - 功能：购物车查看与修改。

- `/orders`  
  - 组件：`WholesaleOrderList`  
  - 功能：订单列表。

- `/orders/:orderId`  
  - 组件：`WholesaleOrderDetail`  
  - 功能：订单详情。

## 三、接口清单（P0 版本草案）

> 说明：以下路径为**建议命名**，真正实现时优先复用现有 WMS/API，只需在接口文档中标注「来源为批发站」。  
> 所有「需要登录」的接口走现有 JWT + 多租户体系。

### 3.1 公共类目与商品接口（未登录可用）

- **GET `/api/public/categories/tree`**  
  - 用途：获取类目树（含 Mobigo 类目）。  
  - 登录要求：否。  
  - 返回字段（示例）：`id`, `name`, `nameEn`, `code`, `parentId`, `children[]`, `sortOrder`。  
  - 备注：应直接复用现有 `GET /categories?tree=true` 能力，新增/别名一个 public 入口即可。

- **GET `/api/public/products`**  
  - 用途：未登录状态下获取「展示用」商品列表（无价格、无库存）。  
  - 参数：  
    - `categoryId`（可选）  
    - `q`（可选，搜索词）  
    - `page`, `pageSize`  
  - 登录要求：否。  
  - 返回字段（示例）：`id`, `name`, `nameEn`, `mainImage`, `brand`, `shortDescription`。  
  - 不返回字段：价格、真实库存、起订量。

- **GET `/api/public/products/:productId`**  
  - 用途：商品详情（无价格/库存版）。  
  - 登录要求：否。  
  - 返回字段：`id`, `name`, `nameEn`, `images[]`, `description`, `specs[]` 等。

### 3.2 登录后商品与价格库存接口

- **GET `/api/wholesale/products`**  
  - 用途：登录后获取完整商品列表。  
  - 参数：同 3.1 的 `products` 接口。  
  - 登录要求：是。  
  - 返回字段（新增）：`wholesalePrice`, `retailPrice`（可选）, `minOrderQty`, `stockStatus`（例如：IN_STOCK/LOW/OUT_OF_STOCK）。  
  - 备注：价格可基于客户等级（tier）从现有 `wholesalePrice` 派生。

- **GET `/api/wholesale/products/:productId`**  
  - 用途：登录后商品详情（含价格/库存/起订量）。  
  - 登录要求：是。

### 3.3 认证与账号接口

- **POST `/api/auth/wholesale/register`**  
  - 用途：零售商注册。  
  - 请求示例：`{ companyName, contactName, email, phone, password }`。  
  - 返回：`userId`, `customerId`, `token`（或要求登录后再返回 token）。  
  - 备注：成功后在现有客户表中生成一条记录，并关联到 tenant。

- **POST `/api/auth/wholesale/login`**  
  - 用途：零售商登录。  
  - 请求示例：`{ email, password }`。  
  - 返回：`token`, `userInfo{ id, name, customerId, tier }`。

> 若你已有统一登录接口，可不新造 `/wholesale/` 前缀，只需约定「此处为零售商角色」即可。

### 3.4 购物车与下单接口

- **GET `/api/wholesale/cart`**  
  - 用途：获取当前用户购物车。  
  - 登录要求：是。  
  - 返回字段：`items[]`，每个包含 `skuId`, `productId`, `name`, `image`, `wholesalePrice`, `qty`, `minOrderQty`。

- **POST `/api/wholesale/cart/items`**  
  - 用途：向购物车新增/更新一条商品。  
  - 请求示例：`{ skuId, qty }`。  
  - 登录要求：是。  
  - 行为：若已存在则更新数量，若为 0 则删除。

- **DELETE `/api/wholesale/cart/items/:skuId`**  
  - 用途：从购物车删除一条商品。

- **POST `/api/wholesale/orders`**  
  - 用途：从购物车创建订单。  
  - 登录要求：是。  
  - 行为：  
    - 校验所有 SKU 的起订量与可用库存；  
    - 创建一笔销售订单写入现有系统（标记 `source='WHOLESALE_SITE'`）；  
    - 锁定库存（沿用当前锁定逻辑）。  
  - 返回：`orderId`。

### 3.5 订单接口

- **GET `/api/wholesale/orders`**  
  - 用途：查询当前登录客户的订单列表。  
  - 参数：`status`（可选）、`page`, `pageSize`。  
  - 返回字段：`id`, `orderNo`, `createdAt`, `status`, `totalAmount`, `currency` 等。

- **GET `/api/wholesale/orders/:orderId`**  
  - 用途：订单详情。  
  - 返回字段：订单信息 + 订单行（SKU、数量、单价、小计）+ 物流基础信息（快递公司、单号等）。

## 四、用户故事与验收标准（P0）

> 下面仅列 P0 必须满足的用户故事。

### 4.1 零售商（未登录）

- **故事 A1：浏览类目与商品（无价格）**  
  - 作为：一个还没注册的零售商  
  - 我想要：从首页或类目树进入，浏览不同类目的商品、看到商品图片和基本描述  
  - 以便：判断这个批发站是否有我感兴趣的品类，再决定是否注册  
  - 验收标准：  
    - 未登录访问 `/`、`/categories/:id`、`/products/:id` 都能正常打开；  
    - 商品卡片和详情页**不显示**任何价格与库存数；  
    - 从首页和类目页都可以点击到详情页。

- **故事 A2：在详情页被引导去登录/注册**  
  - 验收标准：  
    - 未登录访问商品详情时，页面有明显的「登录后查看价格」或「注册」按钮；  
    - 点击按钮可跳转到 `/login` 或 `/register`。

### 4.2 零售商（已登录）

- **故事 B1：登录后可见价格与库存状态**  
  - 验收标准：  
    - 登录成功后再访问商品列表和详情时，能看到批发价和库存状态；  
    - 若某 SKU 无库存，前端有清晰提示，不允许加入购物车（或限制最大数量）。

- **故事 B2：加入购物车与修改数量**  
  - 验收标准：  
    - 在商品列表或详情点击「加入购物车」后，`/cart` 中能看到该商品行；  
    - 修改数量并保存后，界面显示正确数量与小计金额；  
    - 删除一行后，该 SKU 不再出现在购物车中。

- **故事 B3：下单生成订单**  
  - 验收标准：  
    - 在购物车点击「提交订单」前，会校验起订量规则（不符合时给出明确错误信息）；  
    - 提交成功后跳转到订单详情页或订单列表页，可看到新订单；  
    - 后端能在现有订单表中查到该订单，且 `source` 标记为 `WHOLESALE_SITE`（或类似字段）。

- **故事 B4：查看历史订单**  
  - 验收标准：  
    - 登录用户访问 `/orders` 时，只能看到自己的订单；  
    - 订单状态与金额与后台一致；  
    - 点击某订单可进入详情页看到订单行。

### 4.3 运营/仓库（通过现有系统）

- **故事 C1：在现有后台中识别批发站订单**  
  - 验收标准：  
    - 后台订单列表可以按 `source`（或其他字段）过滤出「批发站来源」订单；  
    - 批发站订单的拣货、出库流程与现有订单一致，无需额外操作。

## 五、关键数据规则（P0 最小集）

### 5.1 登录可见与未登录可见

- 未登录：  
  - 可见：类目树、部分商品列表、部分商品详情（无价无库存）。  
  - 不可见：价格、真实库存数量、起订量、购物车、订单。
- 已登录：  
  - 可见：全量商品、批发价、库存状态、起订量、购物车、订单。

### 5.2 起订量规则（P0 建议）

- 建议 P0 先采用**按 SKU 维度**的起订量 `minOrderQty`：  
  - 加入购物车/提交订单时，数量必须为 `minOrderQty` 的整数倍，或至少大于等于 `minOrderQty`。  
  - 若未来有「整单起订金额/数量」等复杂规则，放到 P1/P2。

### 5.3 简化的库存展示

- P0 不要求展示精确数字库存，只要求：  
  - `IN_STOCK`：有库存，可下单；  
  - `LOW_STOCK`：库存较少，前端可展示「库存紧张」文案；  
  - `OUT_OF_STOCK`：无库存，不允许下单。  
- 具体阈值（例如 `< 10` 视为 `LOW_STOCK`）可在后端配置。

### 5.4 埋点（仅占坑）

- P0 阶段只需在前端埋点：  
  - 类目点击（categoryId）  
  - 商品详情打开（productId）  
  - 加入购物车点击（skuId, qty）  
- 这些数据可先写入简单日志或队列表，P0 不做完整报表。

## 六、与现有系统的约定

- 类目与商品：  
  - 直接复用现有 Category + Product/SKU 表与接口；  
  - Mobigo 导入的类目在类目树中正常展示。
- 客户与用户：  
  - 注册成功后，在现有客户表中创建一条记录，并与用户账号关联；  
  - 订单中的客户信息引用这条客户记录。
- 订单：  
  - 通过现有创建销售订单 API 写入；  
  - 拣货、出库等流程完全沿用当前逻辑；  
  - 只需额外标记「订单来源为批发站」字段。

## 七、实现节奏建议

- **第 1 步**：打通认证与类目/商品公共接口（首页+类目+列表+详情，先不下单）。  
- **第 2 步**：实现购物车与下单，确保订单正确写入现有系统。  
- **第 3 步**：补完订单列表/详情页面。  
- **第 4 步**：补基础埋点（类目/详情/加购），为后续 P1 报表做准备。
