## 一、项目背景与目标

### 1.1 业务背景

**[业务填写]** 用 2～3 句话说明：为什么需要批发站、当前痛点、期望达成的业务结果。

- 当前状况：  
  最开始是从深圳散货进口到爱尔兰，再跑到零售商那里去推销，经过 10 年的发展，现在已经是每个月进口 2-6 个货柜到爱尔兰，全部 3000 家零售商都可能来采购。之前没有这个批发站，只能靠电话去描述产品，或者通过网上的某个图片来描述产品，然后再沟通价格，形成采购。而且零售商可能问完价格去别的地方去采购了
  <!-- 例：仅有仓库/进销存系统，零售商通过电话/邮件下单，无法自助浏览与下单 -->
- 期望结果：  
  期望每一个零售商都使用这套系统，这样就能获得第一手的采购数据，采购意图，如果未来还可以推动零售商把他们的销售系统也打通，那就能让数据更加完整了
  <!-- 例：零售商可自助浏览商品、看价、下单；公司可基于行为与订单数据指导补货与进口 -->

### 1.2 产品目标（可量化）

| 目标项 | 描述 | 可选指标 |
|--------|------|----------|
| **[业务填写]** | 例：提升零售商自助下单占比 | 例：上线后 3 个月内占比达 80% |
| **[业务填写]** | 例：沉淀采购意图与销量数据 | 例：每日/周报表可产出 |

## 二、角色与用户场景

### 2.1 角色定义

| 角色 | 说明 | 使用场景（简要） |
|------|------|------------------|
| **零售商（未登录）** | 访问批发站、浏览类目与商品，尚未注册/登录 | 了解品类、比价意向、部分商品展示 |
| **零售商（已登录）** | 已注册的 B2B 客户，可看价格、库存、下单 | 选品、加购、下单、查订单 |
| **运营/销售** | 公司内部，管理商品、价格、活动、客户 | 上下架、改价、看报表、处理异常订单 |
| **仓库/采购** | 使用现有 WMS/进销存 | 接收批发站订单、拣货出库；根据销量与意图做补货/进口 |

**[业务填写]** 如需更多角色（如：客服、财务），需要客服这个角色，可以去订单，查报价，查出库，查物流等。

### 2.2 核心用户场景（按角色）

**零售商（未登录）**

- 场景 1：**[业务填写]** 例：从首页或类目进入，浏览某类目下的商品列表（无价格）
- 场景 2：**[业务填写]** 例：搜索某关键词，看到匹配商品（无价格），点击进入详情
- 场景 3：**[业务填写]** 例：在详情页点击「申请查看价格」或「注册/登录」以继续

**零售商（已登录）**

- 场景 4：**[业务填写]** 例：登录后看到全量类目与商品、批发价、库存、起订量
- 场景 5：**[业务填写]** 例：加购、修改数量、满足起订量后提交订单
- 场景 6：**[业务填写]** 例：在「我的订单」中查看状态、历史订单

**运营/销售**

- 场景 7：**[业务填写]** 例：在后台查看采购意图报表、销量排行，用于选品或清仓决策
- 场景 8：**[业务填写]** 例：配置首页推荐、清仓区、起订量等

**[业务填写]** 其他重要场景可继续列举。

## 四、核心业务流程

### 4.1 零售商从浏览到下单

```
[业务填写] 请用简短步骤描述，例如：
1. 未登录访问首页 → 点击类目/搜索 → 看到部分商品（无价）
2. 点击「登录/注册」→ 完成注册或登录
3. 再次浏览 → 可见价格、库存、起订量
4. 加购 → 满足起订量 → 提交订单
5. 订单进入仓库系统 → 拣货、出库（沿用现有流程）
```

```
- 补货或进口决策是人工根据报表操作，还是系统给出「建议数量」？
```

## 五、数据口径与统计定义

### 5.1 采购意图相关

| 指标 | 定义 | 数据来源 | 备注 |
|------|------|----------|------|
| **类目/商品浏览量** | **[业务填写]** 例：UV 或 PV，按类目/商品聚合 | 前端埋点 or 服务端日志 | |
| **加购意向** | **[业务填写]** 例：点击「加入购物车」或「想采购」按钮 | 按钮埋点 + 登录态（若已登录可关联客户） | |
| **搜索词** | **[业务填写]** 例：用户输入的搜索关键词及次数 | 搜索接口日志 | |
| **高意向未转化** | **[业务填写]** 例：已登录且多次加购但 N 天内未下单 | 需定义「多次」和「N 天」 | |

### 5.2 销量相关

| 指标 | 定义 | 数据来源 | 备注 |
|------|------|----------|------|
| **销量** | **[业务填写]** 例：订单行数量（件数）或订单金额 | 现有销售订单表 | 按日/周/月聚合 |
| **畅销品排行** | **[业务填写]** 例：按销量降序 Top N，按类目或全局 | 订单明细汇总 | 与 3.1 中「销售排行」一致 |

### 5.3 补货/选品建议（若做）

| 指标/规则 | 定义 | 备注 |
|-----------|------|------|
| **高意向+高销** | **[业务填写]** 例：浏览量与销量均高于阈值 → 建议加大进口/安全库存 | |
| **高意向+低销** | **[业务填写]** 例：排查价格、起订量、到货周期 | |
| **低关注** | **[业务填写]** 例：减少进口或列入清仓 | |
