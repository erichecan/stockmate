# StockFlow（StockMate）产品功能说明

> 面向客户的核心业务功能概览  
> 更新日期：2026-02-28

---

## 一、产品定位

**StockFlow** 是一款面向批发商的**全链路库存与进销存管理系统**，覆盖商品主数据、采购、仓储、销售全流程，支持多租户、多仓库、条码作业与批发分级定价。

适用场景：手机配件、消费电子、日用品等**批发贸易企业**，需要管理多品类 SKU、多仓库、采购到货、库存调拨及批发销售订单。

---

## 二、功能模块总览

| 大模块 | 子功能 | 核心能力 |
|-------|--------|----------|
| **1. 商品管理** | 分类、品牌、产品、SKU | 商品主数据与定价体系 |
| **2. 采购管理** | 供应商、采购单、 shipment、收货 | 从下单到到货的全流程 |
| **3. 仓储管理** | 仓库、货位、库存、库存变动 | 多仓、货位、库存账与流水 |
| **4. 销售管理** | 客户、销售订单、拣货、出库 | 批发订单与分级定价 |
| **5. 工具与平台** | 条码打印、多租户、权限 | 作业支撑与账号体系 |

## 三、模块一：商品管理

### 3.1 分类（Categories）

- **树形分类**：支持多级分类（如：手机壳 → iPhone 壳 → iPhone 16 Pro Max 壳）
- **中英双语**：名称、英文名，便于跨境或外贸场景
- **编码**：唯一分类编码，便于系统与报表引用

### 3.2 品牌（Brands）

- **品牌主数据**：品牌名称、编码、Logo
- **与产品关联**：产品可绑定品牌，用于筛选与报表

### 3.3 产品（Products）

- **SPU 维度**：标准产品单位，如「iPhone 16 Pro Max 透明硅胶壳」
- **状态**：草稿、预售、在售、停售
- **属性**：所属分类、品牌、中英文名称与描述、图片

### 3.4 SKU（Stock Keeping Unit）

- **SKU 维度**：具体规格与条码，如「AP-CASE-BLK-01」
- **自动编码**：系统按规则生成唯一 SKU 编码
- **价格体系**：
  - **成本价**（costPrice）：采购成本
  - **批发价**（wholesalePrice）：批发基准价
  - **零售价**（retailPrice）：零售参考价
- **其他属性**：条码、规格（颜色/尺寸等）、重量、图片

## 四、模块二：采购管理

### 4.1 供应商（Suppliers）

- **供应商档案**：名称、编码、联系人、电话、地址
- **采购参数**：付款条款、交货周期（Lead Time）
- **启用/停用**：支持软删除与状态管理

### 4.2 采购单（Purchase Orders）

- **创建采购单**：选择供应商、SKU、数量、单价，自动汇总金额
- **状态流转**：草稿 → 已确认 → 已发货 → 在途 → 已到货 → 已完成 / 已取消
- **采购单明细**：SKU、数量、单价、已收数量

### 4.3  shipment（Shipments）

- **装箱/出运跟踪**：与采购单关联，记录集装箱号、船名、ETD/ETA
- **装箱单**：按箱号记录 SKU 与数量，支持外箱条码
- **用途**：海运/空运追踪，到货后扫码收货核对

### 4.4 收货（Purchase Receipts）

- **收货单**：针对采购单创建收货记录，录入每个 PO 明细的实收数量
- **差异处理**：支持短少、多收、残损、错发等差异类型
- **更新 PO**：自动更新采购单明细的已收数量

## 五、模块三：仓储管理

### 5.1 仓库（Warehouses）

- **多仓库**：主仓、备货仓、FBA 备货中心等
- **属性**：编码、名称、地址、是否默认仓
- **与库存、销售、采购关联**：所有库存与订单均按仓库维度管理

### 5.2 货位（Bin Locations）

- **货位结构**：区-通道-架-位（如 `A-01-01-01`）
- **条码**：货位有唯一条码，用于 PDA 上架、拣货扫码
- **多仓支持**：每个仓库可维护多货位，也可使用无货位（暂存区）

### 5.3 库存（Inventory）

- **维度**：SKU × 仓库 × 货位（货位可为空表示暂存）
- **数量**：总数量、锁定数量，**可用数量 = 总数量 - 锁定数量**
- **库存流水**：所有变动写入不可变流水账（Ledger），便于追溯与对账

### 5.4 库存变动（Operations）

| 操作 | 说明 | 典型场景 |
|------|------|----------|
| **入库（Inbound）** | 增加库存 | 采购到货上架、退货入库 |
| **出库（Outbound）** | 减少库存 | 销售发货、报废、调出 |
| **调拨（Transfer）** | 仓库间移库 | 主仓→备货仓、仓间调拨 |
| **调整（Adjustment）** | 盘盈盘亏 | 盘点、纠错 |
| **锁定（Lock）** | 锁定可用数 | 销售订单确认时预留 |
| **解锁（Unlock）** | 释放锁定 | 订单取消时释放 |
| **退货入库（Return）** | 客户退货 | 退货上架 |

### 5.5 条码与作业

- **SKU 条码**：商品识别，打印商品标签
- **货位条码**：位置识别，打印货位标签，用于上架、拣货
- **外箱条码**：收货用，箱号 + SKU 编码，扫码核对数量并生成入库任务

## 六、模块四：销售管理

### 6.1 客户（Customers）

- **客户档案**：公司名称、编码、联系人、电话、地址
- **客户分级**：NORMAL / SILVER / GOLD / VIP
- **分级定价**：批发价按客户等级打折
  - NORMAL：100% 批发价
  - SILVER：98%
  - GOLD：95%
  - VIP：90%
- **与销售订单关联**：创建订单时自动带出对应单价

### 6.2 销售订单（Sales Orders）

- **创建订单**：选择客户、发货仓库、SKU、数量
- **自动定价**：按客户 tier 自动计算单价与总金额
- **状态流转**：待确认 → 已确认 → 拣货中 → 已打包 → 已发货 → 已完成 / 已取消
- **来源**：支持手工创建，预留 Shopify 等电商来源

### 6.3 订单与库存联动

- **确认订单**：锁定对应 SKU × 仓库的可用库存
- **取消订单**：释放锁定
- **出库确认（Fulfill）**：扣减库存、写入流水、更新订单状态

### 6.4 拣货单（Pick List）

- **生成拣货单**：按货位排序的 SKU 清单，便于按库位顺序拣货
- **打印**：可打印拣货单，配合 PDA 扫码核对
- **拣货数量**：支持记录已拣数量，用于进度跟踪

## 七、模块五：工具与平台

### 7.1 条码打印（Barcode）

- **Code 128**：商品条码、货位条码、外箱条码
- **QR 码**：二维码生成
- **输出**：PNG 图片或 Base64 Data URL，可对接热敏打印机

### 7.2 多租户（Multi-tenancy）

- **租户隔离**：数据按公司/组织隔离，同一系统可服务多家企业
- **租户属性**：名称、域名标识（slug）、套餐、状态
- **用户归属**：用户属于单一租户，登录时按邮箱推断租户

### 7.3 权限与角色

- **角色**：SUPER_ADMIN / ADMIN / PURCHASER / WAREHOUSE / SALES / VIEWER
- **基于角色**：不同模块、操作可按角色控制（如采购员、仓管、销售）
- **用户管理**：用户增删改、启用停用、最后登录时间

### 7.4 仪表盘

- **核心统计**：产品数、SKU 数、分类、品牌、供应商、客户、仓库、采购单、待处理销售订单、库存记录数
- **快捷入口**：跳转到各业务模块

## 八、业务流程串联

### 8.1 采购到货流程

```
采购单 (PO) → Shipment（装箱）→ PackingList（外箱条码）
  → 收货（Purchase Receipt，录入实收与差异）
  → 入库（Inventory Inbound，指定仓库与货位，上架）
```

### 8.2 批发销售流程

```
创建销售订单（选客户、SKU、数量）→ 确认订单（锁定库存）
  → 生成拣货单（按货位排序）→ 拣货、打包
  → 出库确认（Fulfill，扣库存、写流水、订单完成）
```

### 8.3 库存调拨流程

```
库存调拨（Transfer）→ 选择源仓库/货位、目标仓库/货位、SKU、数量
  → 系统扣减源端库存、增加目标端库存，并写入流水
```

## 九、技术特性（面向客户简介）

- **多租户 SaaS**：数据按租户隔离，支持多公司共用一套系统
- **多仓库 + 货位**：支持多仓、库位管理，适配 PDA 扫码作业
- **条码全流程**：商品、货位、外箱条码覆盖采购收货与销售拣货
- **库存流水账**：所有库存变动可追溯，便于对账与审计
- **批发分级定价**：客户等级自动影响销售单价
